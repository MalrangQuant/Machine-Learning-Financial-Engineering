Odds (승산)¶

  • 위 그림 (b)에서 합격에 대한 확률을 p라고 했을때 odds는 아래와 같이 표현된다.
$$ \begin{equation} Odds = \frac{p}{1-p} \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \end{equation} $$
  • 식을 잘 살펴보면, 성공 확률(p)을 실패 확률(1 - p)로 나눈 표현임을 알 수 있다. 즉, odds란 성공 확률이 실패 확률보다 몇 배인지 판단할 수 있는 지표이다. 예를들어 성공 확률이 0.5 (p = 0.5)인 경우 실패 확률 역시 0.5 (1 - p = 1 - 0.5 = 0.5)로 odds는 1이 되며, 만약 성공 확률이 0.8 (p = 0.8)인 경우 실패 확률은 0.2 (1 - p = 1 - 0.8 = 0.2)로 odds는 4가 된다.
  • odds를 이진형 변수를 예측하는 회귀 모형에 적용해보면 아래와 같다.
$$ \begin{equation} \frac{p}{1-p} = \beta_{0} + \beta_{1}x_{1} + \beta_{2}x_{2} + \cdots + \beta_{p}x_{p} + \epsilon \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \end{equation} $$
  • 종속변수가 odds로 표현됨으로써 예측 결과가 odds값 1을 기준으로 1보다 크면 합격, 1보다 작으면 불합격의 형식으로 판단할 수 있을 것이다. 하지만 여기서 딱 한가지 아쉬운 점이 있다. 성공 확률 p는 확률값으로 0과 1 사이의 값으로 표현되므로 모형 좌측의 odds는 결국 0부터 $\infty$까지의 범위를 표현할 수 있는 반면, 우측의 선형 모형은 $-\infty$부터 $\infty$까지의 범위를 표현하기 때문에 좌변과 우변의 범위가 일치되지 않는 것이다.

Odds 범위의 확장 (로지스틱 회귀의 완성)¶

  • 그렇다면 odds가 표현할 수 있는 범위를 $-\infty$까지 확장시킬 수 있는 모형이 있다면 위의 모형보다 더 적절할 것이다. 해결 방법은 간단하다. odds에 log를 취하는 것이다.
$$ \begin{equation} log\left(\frac{p}{1-p}\right) = \beta_{0} + \beta_{1}x_{1} + \beta_{2}x_{2} + \cdots + \beta_{p}x_{p} + \epsilon \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \end{equation} $$
  • log함수는 0부터 $\infty$까지의 값을 정의역(진수)으로 사용하는 함수인데 odds가 표현할 수 있는 값이 딱 log함수의 정의역 범위와 일치한다. 또한 log함수의 치역은 $-\infty$부터 $\infty$까지 표현되어 우리가 원하는 $-\infty$까지 값이 확장될 수 있다.
  • 위와같이 odds에 log를 취한 Log Odds를 종속변수로 활용한 모형을 로지스틱 회귀(Logistic Regression)이라고 한다.
  • 로지스틱 회귀모형을 p에대해 정리해 아래와 같이 표현하면 이를 시그모이드(sigmoid) 함수라고 하며 이는 결국 우리가 찾던 S자 함수가 된다.
$$ \begin{equation} y = \frac{e^{\beta_{0} + \beta_{1}x_{1} + \beta_{2}x_{2} + \cdots + \beta_{p}x_{p} + \epsilon}}{1+e^{\beta_{0} + \beta_{1}x_{1} + \beta_{2}x_{2} + \cdots + \beta_{p}x_{p} + \epsilon}} \ \ \ or \ \ \ y = \frac{1}{1+e^{-(\beta_{0} + \beta_{1}x_{1} + \beta_{2}x_{2} + \cdots + \beta_{p}x_{p} + \epsilon)}}\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \end{equation} $$

  • 로지스틱 회귀에서 역시 $\beta$값을 추정해야 하는데 이때 최대 우도 추정법(Maximum liklihood estimation, MLE)을 사용하게 된다. 사실 앞서 선형 회귀에서 $\beta$값을 구할 때 OLS방법을 통해 설명했는데, 선형 회귀에서도 MLE 방식을 통해 $\beta$값을 구할 수 있다.(이는 수리통계학 수준의 내용을 다루므로 간단히 구두로 설명한다.) 단, 선형 회귀와 로지스틱 회귀 $\beta$값 추정에 있어 차이점은 선형 회귀는 $\beta$값에 대한 정확한 해(exact solution)가 존재하는 반면 로지스틱 회귀는 $\beta$ (beta)값에 대한 정확한 해를 찾을 수 없는 것이 차이점이다. 로지스틱 회귀에서는 이를 보안하기 위해 기울기 하강(Gradient Descent)과 같은 방법을 사용한다.(여기서는 아래 그림을 통해 간단히 설명하며, 더 자세한 내용은 '수치해석학' 과목을 통해 접할 수 있다.)

독립변수 (세포 특징)¶

  • radius : 반경 (중심에서 외벽까지 거리들의 평균값)
  • texture : 질감 (Gray-Scale 값들의 표준편차)
  • perimeter : 둘레
  • area : 면적
  • smoothness : 매끄러움 (반경길이의 국소적 벼노하)
  • compactness : 조그만 정도 (둘레^2/면적-1)
  • concavity : 오목함(윤관의 오목한 부분의 정도)
  • concave points : 오목한 점의 수
  • symmetry : 대칭
  • fractal dimension : 프랙탈 차원 (해안선근사-1)

종속변수 (양성 여부)¶

  • malignant : 0, 악성 / Benign : 1, 양성

# Y_predict를 threshold를 기준으로 0과 1로 나누어주는 함수 생성
def cut_off(y,threshold):
    Y = y.copy() # copy함수를 사용하여 이전의 y값이 변화지 않게 함
    Y[Y>threshold]=1
    Y[Y<=threshold]=0
    return(Y.astype(int))

# 임계값에 따른 비교
def perf_eval(cm):
    # Accuracy
    Accuracy = (cm[0, 0] + cm[1, 1]) / sum(cm.reshape(-1, ))
    # Precision
    Precision = cm[1, 1] /sum(cm[:, 1])
    # Recall
    Recall = cm[1, 1] / sum(cm[1])
    # F1-score
    F1 = 2 * Recall * Precision / (Recall + Precision)
    return ([Accuracy, Precision, Recall, F1])

def plot_ROC(y_train_true, y_train_prob, y_test_true, y_test_prob):
    '''
    a funciton to plot the ROC curve for train labels and test labels.
    Use the best threshold found in train set to classify items in test set.
    '''
    fpr_train, tpr_train, thresholds_train = roc_curve(y_train_true, y_train_prob, pos_label =True)
    sum_sensitivity_specificity_train = tpr_train + (1-fpr_train)
    best_threshold_id_train = np.argmax(sum_sensitivity_specificity_train)
    best_threshold = thresholds_train[best_threshold_id_train]
    best_fpr_train = fpr_train[best_threshold_id_train]
    best_tpr_train = tpr_train[best_threshold_id_train]
    y_train = y_train_prob > best_threshold

    cm_train = confusion_matrix(y_train_true, y_train)
    acc_train = accuracy_score(y_train_true, y_train)
    auc_train = roc_auc_score(y_train_true, y_train)

    print('Train Accuracy: %s ' %acc_train)
    print('Train AUC: %s ' %auc_train)
    print('Train Confusion Matrix:')
    print(cm_train)

    fig = plt.figure(figsize=(10,5))
    ax = fig.add_subplot(121)
    curve1 = ax.plot(fpr_train, tpr_train)
    curve2 = ax.plot([0, 1], [0, 1], color='navy', linestyle='--')
    dot = ax.plot(best_fpr_train, best_tpr_train, marker='o', color='black')
    ax.text(best_fpr_train, best_tpr_train, s = '(%.3f,%.3f)' %(best_fpr_train, best_tpr_train))
    plt.xlim([0.0, 1.0])
    plt.ylim([0.0, 1.0])
    plt.xlabel('False Positive Rate')
    plt.ylabel('True Positive Rate')
    plt.title('ROC curve (Train), AUC = %.4f'%auc_train)

    fpr_test, tpr_test, thresholds_test = roc_curve(y_test_true, y_test_prob, pos_label =True)

    y_test = y_test_prob > best_threshold

    cm_test = confusion_matrix(y_test_true, y_test)
    acc_test = accuracy_score(y_test_true, y_test)
    auc_test = roc_auc_score(y_test_true, y_test)

    print('Test Accuracy: %s ' %acc_test)
    print('Test AUC: %s ' %auc_test)
    print('Test Confusion Matrix:')
    print(cm_test)

    tpr_score = float(cm_test[1][1])/(cm_test[1][1] + cm_test[1][0])
    fpr_score = float(cm_test[0][1])/(cm_test[0][0]+ cm_test[0][1])

    ax2 = fig.add_subplot(122)
    curve1 = ax2.plot(fpr_test, tpr_test)
    curve2 = ax2.plot([0, 1], [0, 1], color='navy', linestyle='--')
    dot = ax2.plot(fpr_score, tpr_score, marker='o', color='black')
    ax2.text(fpr_score, tpr_score, s = '(%.3f,%.3f)' %(fpr_score, tpr_score))
    plt.xlim([0.0, 1.0])
    plt.ylim([0.0, 1.0])
    plt.xlabel('False Positive Rate')
    plt.ylabel('True Positive Rate')
    plt.title('ROC curve (Test), AUC = %.4f'%auc_test)
    plt.savefig('ROC', dpi = 500)
    plt.show()

    return best_threshold
